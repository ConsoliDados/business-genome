# Skill — scenarios

> Chapter of [[README|business-plan]]. **CLI:** run for the scenarios chapter. **Web-chat:** paste.

**Writes:** the *Scenarios* section of `…/strategy/business-plan.md`

## Steps
1. **Pessimistic / Base / Optimistic** — vary the key drivers (demand, pricing, ramp, churn,
   cost) off the [[financial-plan]] base case.
2. **Sensitivity** — which single variable breaks the case? What's the threshold?
3. State the trigger that would move you between scenarios.

## Guard-rails
- Scenarios derive from the financial base case — keep them consistent, not arbitrary.

---
## Prompt (copy-paste)
> Build pessimistic/base/optimistic scenarios off the financial base case (vary demand, pricing,
> ramp, cost), plus a sensitivity analysis identifying the variable that breaks the case and its
> threshold.
